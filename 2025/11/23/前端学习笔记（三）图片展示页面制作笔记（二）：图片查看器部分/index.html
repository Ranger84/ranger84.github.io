前端学习笔记（三）图片展示页面制作笔记（二）：图片查看器部分

点击这里查看代码解析

class ImageViewer {
    constructor() {
        this.currentIndex = 0;
        this.images = [];
        this.init();
    }


init() {
    this.createOverlay();
    this.bindEvents();
}

createOverlay() {
    const overlayHTML = `
        <div class="image-overlay" id="imageOverlay">
            <button class="close-btn">&times;</button>
            <button class="nav-btn prev-btn">&lt;</button>
            <button class="nav-btn next-btn">&gt;</button>
            <div class="loading">加载中...</div>
            <img class="zoomed-image" id="zoomedImage" src="" alt="">
            <div class="image-info" id="imageInfo"></div>
        </div>
    `;
    document.body.insertAdjacentHTML('beforeend', overlayHTML);

    this.overlay = document.getElementById('imageOverlay');
    this.zoomedImage = document.getElementById('zoomedImage');
    this.imageInfo = document.getElementById('imageInfo');
    this.closeBtn = document.querySelector('.close-btn');
    this.prevBtn = document.querySelector('.prev-btn');
    this.nextBtn = document.querySelector('.next-btn');
    this.loading = document.querySelector('.loading');
}

bindEvents() {
    // 关闭按钮
    this.closeBtn.addEventListener('click', () => this.close());

    // 导航按钮
    this.prevBtn.addEventListener('click', () => this.showPrev());
    this.nextBtn.addEventListener('click', () => this.showNext());

    // ESC键关闭
    document.addEventListener('keydown', (e) => {
        if (e.key === 'Escape') this.close();
        if (e.key === 'ArrowLeft') this.showPrev();
        if (e.key === 'ArrowRight') this.showNext();
    });

    // 点击背景关闭
    this.overlay.addEventListener('click', (e) => {
        if (e.target === this.overlay) this.close();
    });
}

open(index, images) {
    this.currentIndex = index;
    this.images = images;
    this.showImage();
    this.overlay.classList.add('active');
    document.body.style.overflow = 'hidden'; // 防止背景滚动
}

close() {
    this.overlay.classList.remove('active');
    document.body.style.overflow = ''; // 恢复滚动
    setTimeout(() => {
        this.zoomedImage.src = '';
    }, 300);
}

showImage() {
    if (this.images.length === 0) return;

    const imageUrl = this.images[this.currentIndex];
    this.loading.style.display = 'block';
    this.zoomedImage.style.display = 'none';

    const img = new Image();
    img.onload = () => {
        this.zoomedImage.src = imageUrl;
        this.zoomedImage.alt = `放大图片 ${this.currentIndex + 1}`;
        this.zoomedImage.style.display = 'block';
        this.loading.style.display = 'none';
        this.updateInfo();
    };
    img.onerror = () => {
        this.loading.textContent = '图片加载失败';
    };
    img.src = imageUrl;
}

showPrev() {
    if (this.images.length === 0) return;
    this.currentIndex = (this.currentIndex - 1 + this.images.length) % this.images.length;
    this.showImage();
}

showNext() {
    if (this.images.length === 0) return;
    this.currentIndex = (this.currentIndex + 1) % this.images.length;
    this.showImage();
}

updateInfo() {
    this.imageInfo.textContent = `${this.currentIndex + 1} / ${this.images.length}`;
}



}

e.target

一个问题:e.target 的返回值是什么？

引申：console.log()的特殊行为和innerHTML的字符串转换机制。
console.log(e.target);在控制台会输出<div class="image-overlay" id="imageOverlay">...</div>
innerHTML 输出会显示[object HTMLDivElement]
核心原因：不同的转换机制

  1. console.log() 的智能显示
    浏览器控制台对 DOM 元素有特殊处理，它会：

  • 识别出这是 DOM 元素对象

  • 直接显示该元素的 HTML 结构

  • 提供交互式查看（可以展开看属性）

  1. innerHTML 的字符串转换
    点击这里查看代码解析
    
element.innerHTML = e.target;

innerHTML 会：

  • 调用 e.target.toString() 方法

  • 得到标准的对象字符串表示：[object HTMLDivElement]

  • 将这个字符串作为 HTML 内容插入

总结
根本原因：

  • console.log(e.target) → 浏览器特殊优化，显示可视化 HTML 结构

  • innerHTML = e.target → 自动调用toString()，得到 [object HTMLDivElement]

  • 设计目的不同：console.log 为调试优化，innerHTML为内容插入设计

总体模块划分&思路解读

这个项目可以清晰地划分为三个核心模块：

  1. 图片资源加载器 (Image Loader)

  2. 画廊渲染器 (Gallery Renderer)

  3. 图片查看器 (ImageViewer)

它们的依赖关系是：1 → 2 → 3。查看器依赖于渲染器生成的 DOM，渲染器依赖于加载器提供的图片 URL。

编写顺序与思路详解

编写顺序基本就是按照模块依赖关系，自底向上进行的。

第一阶段：搭建基础 - 图片资源加载器
目标： 解决“如何可靠地获取到图片地址”这个核心问题。

  1. 编写checkImageExists(url)函数：

    • 思路： 这是所有功能的基石。需要一种方法在 JavaScript 中探测一个图片 URL 是否有效。

    • 技术选型： 利用Image对象的onload和onerror事件。为了便于使用，将其包装成一个Promise。这样后续可以用async/await以同步的方式写异步代码，更清晰。

    • 注意： 这个函数本身不关心图片格式或编号，它只负责测试一个给定的 URL。

  2. 编写getImageURL(number)函数：

    • 思路： 对于第n张图，我们不知道它是什么格式。所以要用上一个函数，按照imageFormats数组的顺序，逐个格式去尝试。

    • 技术实现： 在一个循环中，拼接出不同格式的 URL，然后用await checkImageExists(testUrl)等待检测结果。一旦某个格式被证实存在，立即返回该 URL。如果所有格式都不存在，则返回undefined。

  3. 在createImageGallery()函数中进行批量检测：

    • 思路： 我们要检测 26 张图，如果一张一张顺序检测，太慢了（下一张要等上一张出结果后才能开始）。

    • 性能优化： 使用Promise.all()。先用一个循环创建 26 个 Promise（每个 Promise 代表对一张图的检测任务），然后让它们并发执行。这样网络请求可以同时发出，极大地缩短了总等待时间。

    • 结果： imageUrls是一个包含了 26 个 URL（或 undefined）的数组，其顺序与图片编号一一对应。

至此，我们已经拿到了所有可用图片的数据。
第二阶段：构建视图 - 画廊渲染器
目标： 将获取到的图片 URL 数据变成用户能看到的图片网格。

  1. 在createImageCard(url)中生成 HTML 字符串：

    • 思路： 数据（imageUrls数组）准备好了，现在要生成视图。在非 React/Vue 的纯 JS 环境中，最直接的方式是拼接 HTML 字符串。

    • 技术实现： 使用array.map()遍历imageUrls，对每个有效的 URL，生成一个.img_card包裹.img_item的 HTML 片段。然后用join('')将它们合并成一个巨大的字符串。

    • 输出： 一个完整的、代表整个画廊的 HTML 字符串。

  2. 编写displayGallery()函数：

    • 思路： 这是主控函数，负责协调数据和视图。

    • 执行流程：await createImageGallery()→ 等待所有图片检测完成并拿到 HTML 字符串 → 将字符串插入到 DOM 容器（gallery.innerHTML = ...）中。

    • 错误处理： 用try...catch包裹整个过程，确保万一出错能有日志可查，不会让页面白屏。

至此，一个静态的图片画廊已经可以正常显示了。
第三阶段：增强交互 - 图片查看器
目标： 为静态画廊添加“点击放大、切换浏览”的交互功能。

  1. 设计ImageViewer类：

    • 为什么用类？ 因为查看器是一个有状态的、功能复杂的独立单元。它需要维护当前索引、图片列表等状态，并包含很多方法。用类可以很好地封装这些状态和行为，比一堆散落的函数和变量要清晰得多。

    • 构造器constructor()： 初始化状态（currentIndex,images）并立即调用init()

  2. 实现ImageViewer的各个方法（按功能逻辑）：

    • createOverlay()： 创建查看器所需的 DOM 结构（遮罩层、关闭按钮、导航按钮、图片元素等），并插入到body末尾。同时将常用 DOM 元素的引用保存在类的属性上（如this.overlay, this.zoomedImage），方便后续操作。

    • bindEvents()： 集中绑定所有事件监听器。

      • 关闭按钮 → close()

      • 导航按钮 → showPrev() / showNext()

      • 键盘事件（ESC, 左右箭头）

      • 点击遮罩背景关闭。

    • open(index, images)： 这是给外部调用的 API。接收一个索引和完整的图片 URL 数组，保存到自身状态，然后调用 showImage() 显示图片，最后将遮罩层设为可见（active）。

    • showImage()： 核心方法。根据currentIndex 从 images中取出 URL，进行预加载。

      • 显示 “加载中…”

      • 创建一个新的 Image 对象去加载图片。

      • onload成功后，再将 URL 设置给真正显示的 this.zoomedImage，并隐藏加载提示。这个过程保证了用户体验的流畅。

    • close(), showPrev(), showNext(), updateInfo()： 这些是相对简单的功能方法，主要工作是更新状态（如 currentIndex）和调用 showImage() 刷新视图。

  3. 将查看器挂接到画廊上：

    • 编写 addImageClickEvents() 函数：

      • 思路： 等画廊渲染完成后，为每个图片卡片绑定点击事件。

      • 技术实现： 获取所有 .img_card 元素，遍历它们。为每个卡片添加点击监听，当点击时，调用 imageViewer.open(index, validImageUrls)。

      • 关键点： validImageUrls 是从已生成的 DOM 中反向收集的所有有效图片的 src，并过滤掉了可能存在的占位符。这保证了查看器里的图片列表和画廊里显示的完全一致。

    • 改造 displayGallery()： 在 innerHTML 执行之后，立即调用 addImageClickEvents()，从而为动态生成的图片元素添加上交互能力。

总结

这个编写顺序体现了经典的分层架构和关注点分离思想：

  1. 先解决数据问题（图片是否存在？URL 是什么？）。

  2. 再解决显示问题（如何把数据变成 HTML？）。

  3. 最后解决交互问题（如何让 HTML 动起来？）。
