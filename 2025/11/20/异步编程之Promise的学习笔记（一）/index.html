异步编程之Promise的学习笔记

遇到难题了……梳理一下……

什么是Promise？

Promise 是异步编程的一种解决方案，是 JavaScript 中用于处理异步操作的对象，它代表了一个异步操作的最终完成（或失败）及其结果值。

Promise 的状态

Promise 有三种状态：

  • Pending（等待）：初始状态，既没有完成也没有失败。
  • Fulfilled（已完成）：操作成功完成。
  • Rejected（已失败）：操作失败。

工作原理

创建 Promise

—javascript
const myPromise = new Promise((resolve, reject) => {
// 执行异步操作
if (/* 操作成功 */) {
resolve(成功数据); // 将状态改为 fulfilled
} else {
reject(错误信息); // 将状态改为 rejected
}
});

使用 Promise

—javascript
myPromise
.then(result => {
// 成功时的处理
console.log(“成功:”, result);
})
.catch(error => {
// 失败时的处理
console.log(“失败:”, error);
})
.finally(() => {
// 无论成功失败都会执行
console.log(“操作结束”);
});

.then() 方法

.then()的作用：当 Promise 成功完成（resolve）时，.then() 中的回调函数会被执行。

基本语法

—javascript
promise.then(
onFulfilled, // Promise 成功时执行的回调函数
onRejected // （可选）Promise 失败时执行的回调函数
);

执行流程

  1. 创建 Promise → 检查 x == 0
  2. 条件判断 → 5 == 0 为 false
  3. 执行分支 → 调用 myReject(“Error”)
  4. 处理结果 → .then() 的第二个函数执行
  5. 显示结果 → 页面上显示 “Error”

问题

1.是否可以将
—javascript
myPromise.then(
function(value) {myDisplayer(value);}, // 成功时执行
function(error) {myDisplayer(error);} // 失败时执行
);

修改为
—javascript
myPromise.then(
myDisplayer(value),
myDisplayer(error)
);

答：不可以直接这样改写。
原始代码
—javascript
myPromise.then(
function(value) {myDisplayer(value);}, // 回调函数
function(error) {myDisplayer(error);} // 回调函数
);

错误改写
—javascript
// 这样是错误的！
myPromise.then(
myDisplayer(value), // ❌ 立即执行函数，不是回调
myDisplayer(error) // ❌ 立即执行函数，不是回调
);

问题：

  • myDisplayer(value) 会立即执行，而不是作为回调函数
  • value 和 error 变量此时不存在，会导致错误

正确的简化写法
方法 1：使用箭头函数（推荐）
—javascript
myPromise.then(
value => myDisplayer(value), // 箭头函数
error => myDisplayer(error) // 箭头函数
);

方法 2：直接传递函数引用
—javascript
myPromise.then(
myDisplayer, // ✅ 直接传递函数引用
myDisplayer // ✅ 直接传递函数引用
);

2.从上述代码可以看出，resolve和reject不是系统关键字，可以自定义命名，在Promise构造函数中，这两个参数的名字可以自定义，如：
—javascript
// 这些写法都是正确的：
let promise1 = new Promise(function(resolve, reject) {
// 使用 resolve 和 reject
});

let promise2 = new Promise(function(success, failure) {
if (condition) {
success(“成功”);
} else {
failure(“失败”);
}
});

let promise3 = new Promise(function(ok, error) {
ok(“完成”);
});

let promise4 = new Promise(function(完成, 拒绝) {
完成(“成功”);
});

只是约定俗成使用resove和reject，这样代码更易读和理解。
这就带我来到下面的问题
3.在上述代码中，myResolve, myReject似乎是两个函数，可是它们没有写出函数定义，但程序依然正常运作，这是为什么？
myResolve和myReject确实没有在代码中明确定义，但它们能正常工作的原因是：
答案：它们是 Promise 构造函数自动提供的
当你创建new Promise()时，JavaScript 引擎在内部自动创建了这两个函数，并把它们作为参数传递给你提供的执行器函数。
也就是：

  • 你：提供”做什么”的逻辑（if-else 判断）
  • JavaScript 引擎：提供”怎么做”的工具（myResolve/myReject 函数）

4.如果我没有定义myResolve函数，系统如何知道myResolve(“OK”)是什么意思呢？
答：Promise 构造函数在内部已经预定义了这两个函数的具体行为。
系统内部：
- myResolve函数：将 Promise 的状态设置为 fulfilled，并将成功数据作为参数传递给回调函数
- myReject函数：将 Promise 的状态设置为 rejected，并将错误信息作为参数传递给回调函数
—javascript
// 当你创建 Promise 时，JavaScript 引擎内部做了这些事情：
class Promise {
constructor(executor) {
// 1. 系统预先定义好 resolve 函数的具体实现
const resolve = (value) => {
// 内部逻辑：将 Promise 状态改为 fulfilled
// 存储 value 值
// 触发所有 .then() 中注册的成功回调
};

// 2. 系统预先定义好 reject 函数的具体实现
const reject = (reason) => {
// 内部逻辑：将 Promise 状态改为 rejected
// 存储 reason 值
// 触发所有 .then() 或 .catch() 中注册的失败回调
};

// 3. 立即执行你传的函数，并把上面两个函数作为参数传给你
executor(resolve, reject);
}
}

具体到上面的代码示例：
—javascript
// 你写的：
let myPromise = new Promise(function(myResolve, myReject) {
let x = 0;
if (x == 0) {
myResolve(“OK”); // ← 这里调用系统提供的函数
}
});

// 实际发生的情况：
let myPromise = new Promise(function(myResolve, myReject) {
let x = 0;
if (x == 0) {
// 调用的是系统内部定义的函数，该函数知道：
// 1. 把 Promise 状态从 pending 改为 fulfilled
// 2. 把 “OK” 存储起来
// 3. 通知所有在 .then() 中等待的回调函数
myResolve(“OK”);
}
});

类比理解：就像我们使用console.log()：
—javascript
console.log(“Hello”); // 你不需要知道 console.log 如何实现

你知道调用console.log()会在控制台输出信息，但不需要知道它内部是怎么实现的。同样，myResolve和myReject是系统提供给你的”工具函数”，你只需要知道：

  • myResolve(value) = 告诉系统：”异步操作成功了，结果是 value”
  • myReject(reason) = 告诉系统：”异步操作失败了，原因是 reason”
    系统在背后处理所有的状态管理、回调触发等复杂逻辑。
    也就是说：
    你可以自定义参数名，但功能和行为已经由系统预设好了。
    核心理解：
  • 名字：你自定义
  • 功能：系统预设（改变 Promise 状态、触发回调等）
    举例说明：
    —javascript
    // 所有这些写法功能完全一样：
    let p1 = new Promise(function(resolve, reject) {
    resolve(“成功”); // 系统知道这是”完成操作”
    });

let p2 = new Promise(function(成功, 失败) {
成功(“成功”); // 系统知道这是”完成操作”
});

let p3 = new Promise(function(done, error) {
done(“成功”); // 系统知道这是”完成操作”
});

let p4 = new Promise(function(a, b) {
a(“成功”); // 系统知道这是”完成操作”
});

5.能否认为：myResolve(“OK”);的逻辑就是当成功时返回字符串OK，myReject(“Error”);的逻辑就是当失败时返回字符串Error，他们的运行逻辑是系统决定好的？
答：不正确，是有差别的。
系统的预设逻辑：
myResolve("OK")的系统逻辑：
将Promise状态从pending改为 fulfilled
将值"OK"存储起来
触发所有通过.then()注册的成功回调函数，并把"OK"传给它们
myReject("Error")的系统逻辑：
将Promise状态从pending改为rejected
将错误原因"Error"存储起来
触发所有通过.then()或.catch()注册的失败回调函数，并把"Error"传给它们
更准确的理解：
—javascript
// 你调用：
myResolve(“OK”);

// 系统内部执行：
// 1. this.state = ‘fulfilled’; (状态改变)
// 2. this.value = “OK”; (存储值)
// 3. 执行所有等待中的成功回调：callback(“OK”)

所以不是”返回”字符串，而是”传递”字符串给后续的回调函数。
6.在问题1的代码简化方法中，代码可以简化写为：
—javascript
myPromise.then(
myDisplayer, // 成功时调用 myDisplayer(result)
myDisplayer // 失败时调用 myDisplayer(error)
);

也就是直接传递函数引用。但是在定义中，myDisplayer函数不是需要一个参数吗？为什么在简写版本中myDisplayer函数不需要参数呢？
答：在简写版本myPromise.then(myDisplayer, myDisplayer)中，参数是自动传递的。
当你说myPromise.then(myDisplayer)时，你是在说：
“当Promise完成时，请调用myDisplayer函数，并且把你得到的结果值作为第一个参数传给它。”
JavaScript 引擎内部的处理相当于：
—javascript
// 当你写：
myPromise.then(myDisplayer);

// JavaScript 实际上执行的是：
myPromise.then(function(result) {
return myDisplayer(result); // 自动把结果值传给 myDisplayer
});

所以
—javascript
// 完整版本：
myPromise.then(
function(value) { myDisplayer(value); }, // 手动传递value
function(error) { myDisplayer(error); } // 手动传递error
);

// 简写版本：
myPromise.then(myDisplayer, myDisplayer); // 自动传递value或error

两种写法完全等效，因为.then()方法会自动把解析值或拒绝原因作为参数传递给指定的回调函数。

7.then()方法中一定会用到回调函数对吗？
是的，完全正确，这是它的核心设计。
.then()的基本结构
—javascript
promise.then(
onFulfilled, // 必需：成功时的回调函数
onRejected // 可选：失败时的回调函数
);

为什么必须用回调函数：
因为Promise是异步的，系统不知道你什么时候调用resolve()/reject()，所以需要你先”注册”好回调函数，等结果准备好时再调用它们。
各种回调函数的写法：
—javascript
// 1. 函数表达式
myPromise.then(
function(value) { console.log(value); },
function(error) { console.error(error); }
);

// 2. 箭头函数
myPromise.then(
value => console.log(value),
error => console.error(error)
);

// 3. 已定义的函数引用
myPromise.then(console.log, console.error);

// 4. 方法调用
myPromise.then(
value => myDisplayer(value),
error => myDisplayer(error)
);

总结：.then()的本质就是”当异步操作完成时，请调用这些回调函数”，所以回调函数是必须的。
